# Contributing to Mekabiz-Autoclicker
Thank you for considering contributing to Mekabiz-Autoclicker, a customizable autoclicker Chrome extension with hotkeys. We appreciate your help in making this project better.

## Getting Started
To contribute to Mekabiz-Autoclicker, follow these steps:

1. **Fork the repository**: Go to the [Mekabiz-Autoclicker repository](https://github.com/your-username/Mekabiz-Autoclicker) and click the "Fork" button. This will create a copy of the repository in your own GitHub account.
2. **Clone the repository**: Run `git clone https://github.com/your-username/Mekabiz-Autoclicker.git` to clone the forked repository to your local machine.
3. **Create a new branch**: Run `git checkout -b feature/your-branch-name` to create a new branch for your changes. See the [Branch Naming](#branch-naming) section for guidelines on naming your branch.

## Branch Naming
When creating a new branch, please follow these naming conventions:

* **Feature branches**: Use the format `feature/your-branch-name` (e.g., `feature/add-hotkey-support`).
* **Bug fix branches**: Use the format `fix/your-branch-name` (e.g., `fix/fix-autoclicker-issue`).
* **Release branches**: Use the format `release/vx.x.x` (e.g., `release/v1.2.3`).

## Committing Changes
When committing changes, please follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification:

* **Format**: Use the format `type(scope): brief description` (e.g., `feat: add hotkey support`).
* **Type**: Use one of the following types:
	+ `feat` for new features
	+ `fix` for bug fixes
	+ `docs` for documentation changes
	+ `style` for code style changes
	+ `refactor` for code refactoring
	+ `perf` for performance improvements
	+ `test` for test additions
	+ `build` for build system changes
	+ `ci` for continuous integration changes
	+ `chore` for miscellaneous changes

## Pull Request Checklist
Before submitting a pull request, please ensure that:

1. **You have forked the repository** and are submitting a pull request from your fork.
2. **Your branch is up-to-date** with the latest changes from the main repository.
3. **Your changes are committed** with a clear and descriptive commit message.
4. **You have tested your changes** thoroughly and verified that they work as expected.
5. **You have updated the documentation** (if necessary) to reflect your changes.
6. **You have included any necessary screenshots or videos** to demonstrate your changes.

By following these guidelines, you can help ensure that your contribution is accepted and makes it into the next release of Mekabiz-Autoclicker. Thank you again for your contribution!
